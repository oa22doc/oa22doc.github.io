Layer (Overlap)

 LAYER layerName

    TYPE OVERLAP ;

    [PROPERTY propName propVal ;] ...

 END layerName

An OVERLAP layer is not mapped as an oaLayer. Instead, the name of the OVERLAP layer is stored using an oaStringProp on the LEF Legacy oaHierProp. Shapes on an overlap layer are merged and stored in the oaPRBoundary. If shapes on a TYPE OVERLAP layer are not contiguous, the macro is saved but the shapes are ignored. See the MACRO OBS documentation.

PROPERTY propName propVal

Properties are not supported because the OVERLAP layer is not stored in OpenAccess.
