The MACRO PIN ANTENNA attributes are stored in OpenAccess using an oaAntennaData object. This oaAntennaData object is stored in the oaTerm::AntennaData attribute. There is an oaAntennaData object for each ANTENNAMODEL used, which is accessed by specifying the right oaAntennaModelEnum when getting and setting the oaAntennaData on the oaTerm. The following table shows the mapping from LEF ANTENNAMODEL to oaAntennaModelEnum.

LEF ANTENNAMODEL oaAntennaModelEnum
OXIDE1 (default) oacDefaultAntennaModel
OXIDE2 oacSecondAntennaModel
OXIDE3 oacThirdAntennaModel
OXIDE4 oacFourthAntennaModel

Each of the ANTENNA*AREA values is stored in OpenAccess by using an oaAntennaArea object, which is then appended to one of the oaAntennaData object's oaAntennaAreaArray attributes as shown in the following table.

ANTENNA value oaAntennaData attribute
ANTENNAPARTIALMETALAREA oaAntennaData::partialMetal
ANTENNAPARTIALMETALSIDEAREA oaAntennaData::partialMetalSide
ANTENNAPARTIALCUTAREA oaAntennaData::partialCut
ANTENNADIFFAREA oaAntennaData::diff
ANTENNAGATEAREA oaAntennaData::gate
ANTENNAMAXAREACAR oaAntennaData::maxCAR
ANTENNAMAXSIDEAREACAR oaAntennaData::maxSideCAR
ANTENNAMAXCUTCAR oaAntennaData::maxCutCAR

DIRECTION {INPUT | OUTPUT [TRISTATE] | INOUT | FEEDTHRU} ;

The MACRO PIN DIRECTION attribute maps to OpenAccess using the oaTerm::TermType attribute. The following table shows the mapping between PIN DIRECTION and the oaTermTypeEnum values.

PIN DIRECTION oacTermTypeEnum
INPUT oacInputTermType (default)
OUTPUT oacOutputTermType
OUTPUT TRISTATE oacTristateTermType
INOUT oacInputOutputTermType
FEEDTHRU oacJumperTermType

MUSTJOIN pinName ;

The MUSTJOIN attribute is stored in OpenAccess using the mustJoin attribute on the oaBitTerm associated with the LEF PIN. If the terminal specified by the mustJoin attribute does not exist in the design, it is an error. MUSTJOIN on POWER and GROUND PINS will be ignored with a warning and the ports will be merged with the ports of the original LEF PIN.

NETEXPR "netExprPropName defaultNetName" ;

The NETEXPR maps to an oaTermConnectDef in OpenAccess. The netExprPropName is stored as the assignmentName of an oaAssignmentDef, and the defaultNetName is stored as the defaultName of the oaAssignmentDef. The oaAssignmentDef is then associated with the current oaTerm by the oaTermConnectDef.

PORT
   [CLASS {NONE | CORE} ;]
   {layerGeometries} ...
 END

The PORT attributes are stored in OpenAccess as oaPin objects. The associated shapes and vias are stored as the oaPin figures. For each port geometry, a new shape is created and attached to the oaPin.

The CLASS attribute maps to an oaStringProp named CLASS on the oaPin representing the LEF PORT. For backward compatibility, an oaBooleanProp named CLASSCORE is also set for CLASS CORE ports.

PROPERTY propName propVal ;

See PROPERTIES.

SUPPLYSENSITIVITY powerPinName ;
 GROUNDSENSITIVITY groundPinName ;

The LEF PIN SUPPLYSENSITIVITY attribute maps to OpenAccess as the supplySensitivity attribute of the current oaTerm. The oaTerm specified by powerPinName must exist in the design.

The LEF PIN GROUNDSENSITIVITY attribute maps to OpenAccess as the groundSensitivity attribute of the current oaTerm. The oaTerm specified by groundPinName must exist in the design.

SHAPE {ABUTMENT | RING | FEEDTHRU} ;

The MACRO PIN SHAPE attribute is stored in OpenAccess using the oaTerm::PinConnectMethod attribute. The following table shows the mapping between PIN SHAPE and the oaPinConnectMethodEnum values.

PIN SHAPE oaPinConnectMethodEnum
none oacPinConnectMethodDefault (default)
ABUTMENT oacPinConnectMethodAbut
RING oacPinConnectMethodRing
FEEDTHRU oacPinConnectMethodFeedthru

TAPERRULE ruleName ;

The MACRO PIN TAPERRULE attribute maps to OpenAccess as a constraint group member for the oaNet attached to the current oaTerm. The oaConstraintGroup named ruleName must exist in the technology database.

USE { SIGNAL | ANALOG | POWER | GROUND | CLOCK } ;

The MACRO PIN USE attribute is stored in OpenAccess using the oaNet::SigType attribute of the oaNet attached to this oaTerm. The following table shows the mapping between PIN USE and the oaSigTypeEnum values.

PIN USE oacSigTypeEnum
SIGNAL oacSignalSigType (default)
ANALOG oacAnalogSigType
CLOCK oacClockSigType
GROUND oacGroundSigType
POWER oacPowerSigType

Note: If the pin is not a power or ground pin, the oaBlock created for the current LEF MACRO is made visible to the module domain.
