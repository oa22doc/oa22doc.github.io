Help Using OpenAccess Documentation

The OpenAccess 2.2 C++ API documentation set includes:

  • API Specification
  • Programmers Guide
  • Information Model
  • OpenAccess Coding Style and Standards
  • Translator documents
  • Installation and Configuration Notes
  • Release Notes
  • Known Problems and Solutions
  • Migration documents
  • Design Management Plug-In API Specification
  • Help

Navigating the Documentation

The API Specification, Programmers Guide, Information Model, and Coding Style and Standards documentation share a common navigation scheme in the form of a navigation bar at the top of each page. The navigation bar includes several icons that link to different organizational views of the OpenAccess reference documentation.

The Programmers Guide navigation bar icon links to a top-level page that lists the Programmers Guide topics. The Information Model navigation bar icon links to a page that lists the information model diagrams in alphabetical order. The Reference, Programmers Guide, and Information Model documentation are interlinked for easy cross-reference.

The Help icon links to this page. From any page in the documentation, you can navigate to any other top-level page using the navigation bar.

Finding API Reference Information

There are several different ways to access the API reference information. In particular, the following three buttons (at the top of each page) provide different views of the API documentation:

  • All Classes
  • Class Categories
  • Index

The All Classes page lists all API classes in alphabetical order.

The Class Categories page organizes the API listings into functional areas, such as oaDesign classes, DM classes, technology classes, and so forth.

The Index page provides links to different OpenAccess all-class indexes, including functions, globals, files, hierarchy, and namespace indexes.

Understanding the Documentation Format

Each API class has a reference page in the documentation set. Just below the navigation bar at the top of each page, you find a link to the header file for the class. For example, on the oaTextDisplay page you find #include <oaTextDisplay.h> just beneath the page title. Near the top of each page there is also an inheritance diagram that can show both ancestor and derived classes. By clicking a shaded box in the diagram, you jump directly to the documentation for that ancestor or derived class.

For added convenience, an "Inherited Members " link is included on each reference documentation page. This link displays an alphabetical listing of all member functions for the class. A list of the methods and attributes available to the class is also part of each page, followed by the detailed description of the class.

The remaining portion of the page includes member function documentation.
