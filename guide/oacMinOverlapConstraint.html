minOverlap (oacMinOverlap)

Description

The built-in layer pair constraint definition oacMinOverlap specifies the minimum distance a shape on one layer must overlap a shape on a second layer in database units.

Overlaps are specified from the inside edge of the shape on the second layer to the inside edge of the shape on the first layer.

This constraint definition is not symmetric, implying that the minimum overlap of layer1 past layer2 is not the same as the minimum overlap of layer2 past layer1.

Constraint at a Glance

Constraint type: oaLayerPairConstraint (Symmetric: no)
Value types: oaIntValue
Database types: oaDesign, oaTech
Object types: oaAppObject

Values

The following value types are supported by this constraint:

  • oaIntValue

    This Value represents the minimum extension in database units.

    Units: DBU

Examples


Manufacturing Reasons

Overlap rules ensure that the necessary geometry overlaps are maintained to provide good connections after manufacturing.
