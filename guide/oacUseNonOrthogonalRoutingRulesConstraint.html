oaUseNonOrthogonalRoutingRules (oacUseNonOrthogonalRoutingRules)

Description

The oacUseNonOrthogonalRoutingRules simple constraint designates whether a block or area of a design is routed using 45 degree and 135 degree routing pitches and offsets. Diagonal routing pitches and offsets are set separately by corresponding constraints.

Constraint at a Glance

Constraint type: oaSimpleConstraint
Value types: oaBooleanValue
Database types: oaDesign
Object types: oaBlock, oaAreaBoundary, oaAppObject

Values

The following value types are supported by this constraint:

  • oaBooleanValue

    oacUseNonOrthogonalRoutingRules uses an oaBoolean value, where true designates that non-orthogonal routing is used.

    Units: Boolean
