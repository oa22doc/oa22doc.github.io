minSameNetSpacing (oacMinSameNetSpacing)

Description

The built-in layer constraint, oacMinSameNetSpacing, specifies the minimum distance required between two electrically equivalent geometries on the same layer. The spacing between electrically equivalent geometries can be dependent on the width of the geometries. See the figure below.

This constraint should be used in conjunction with oacMinSpacing (different net spacing) and should only be used when the same net spacing is smaller than the different net spacing.

Constraint at a Glance

Constraint type: oaLayerConstraint
Value types: oaIntValue, oaInt1DTblValue
Database types: oaDesign, oaTech
Object types: oaAppObject

Values

The following value types are supported by this constraint:

  • oaIntValue

    This oaIntValue represents the minimum spacing required between the two geometries in database units.

    Units: DBU

  • oaInt1DTblValue

    This oaInt1DTblValue that specifies spacing values based on width.

    Units: DBU

Examples


Manufacturing Reasons

This constraint is used to eliminate notch filling in certain situations. When the minimum same net spacing constraint is present, the notch only needs to be filled if the gap is smaller than the value of the constraint.
