Library Server

The library server manages and controls concurrent access to a library. OpenAccess cannot access a library unless there is a library server running for that library.

A library server can be started manually or automatically. When you manually start the library server on a host machine, you can specify a port number that clients will use to communicate with the server. The default port number is 55555. When the library server first starts, no libraries are under its control. Clients connect to the library server via a TCP socket and provide the hostname and port number. Clients request that the library server provide access to existing libraries or create new ones.

A client can query and control the library server through the oaLibServer class.

An OpenAccess client application specifies the library server to be used for the library through the library definition (oaLib::addDef). The server specification is used to automatically start a library server if an application attempts to access a library using oaLib::find or create a library using oaLib::create and no server is currently running.

Only one library server can be active on a library at a time. The server access is controlled by a lib.server file, which is created in the directory containing the library. If this file is present, it contains the hostname and the port number of the current server for the library. The lib.server file also serves as a lock file that prevents other library servers from accessing the library.

You can run multiple library servers on a host machine as long as each server uses a different port name. A library server can be used to serve more than one library.

Run the library server from a command prompt.

UNIX Windows
oaLibServer oaLibServer.exe

Argument Descriptions

[-portNumber number]
 Specifies the port number for communicating with a client.
Valid values: Numbers greater than 1023 that are not being used by another server on the host machine.
Default value: 55555 
[-logFile file]

Specifies the name of the log file.
Default value: server.slog
